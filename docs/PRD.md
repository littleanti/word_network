## 1. 제품 개요

**제품명**: 순우리말 어휘망 연결 — 그림 보고 음절 잇기

**한 줄 요약**: 동화책·일상 장면 일러스트를 보고, 흩어진 음절 블록을 순서대로 연결하여 알맞은 고유어 단어를 완성한 뒤 그림 속 객체에 매칭하는 모바일 우선 학습 게임.

**위치**: `hangul_games` 7단계 로드맵의 **3단계** — `2_syllable_assembly`(음절 조립)에서 익힌 자모/음절 단위 읽기를, **수평적 어휘 인출**로 자동화하는 단계.

**배포 환경**: 단일 페이지 정적 웹앱 (모바일 웹 우선 / PWA 후보) · 포트 **4323**

## 2. 인지·교육 목표

| 항목 | 내용 |
|------|------|
| 대상 연령 | 만 5 ~ 7세 (유아 후기 ~ 초등 1학년) |
| 발달 단계 | 어휘 확장 + 수평적 읽기 자동화 |
| 핵심 인지 목표 | ① 시각적 맥락 ↔ 어휘 매칭, ② 음절 시퀀스 조립, ③ 체언(명사) 우세 어휘 발달 |
| 부차 목표 | 청각-시각 통합(TTS), 한국어 고유어 친숙도, 한자어 진입 전 일상어 자동화 |
| 선행 게임 | `2_syllable_assembly/` — 자모·음절 결합 능력 보유 |
| 후행 게임 | `4_morpheme_detective/` — 단어 속 의미 단위 인지로 진입 |

## 3. 타겟 사용자

| 사용자 그룹 | 특징 | 주요 니즈 |
|---|---|---|
| 유아 후기 ~ 초1 | 음절은 읽지만 단어로 통합 자동화는 미완성 | 그림 단서, 큰 터치 타겟, 즉각 칭찬 피드백 |
| 학부모 | 옆에서 함께 놀이 | 코칭 프롬프트, 난이도/카테고리 조절, 오답 복습 |
| 어린이집·유치원 교사 | 그룹 학습 보조 | 카테고리 한정, 세션 길이 조절, 정답률 확인 |
| 한국어 학습자(외국인 가족) | 고유어 어휘 노출 | TTS 발음, 카테고리별 어휘 확장 |

## 4. 사용자 시나리오

### S1. 아이가 혼자 놀기
- 부모가 태블릿을 쥐여주고 "동물원 갔다 왔지? 이거 해볼래?" 한 마디 후 자리를 뜸
- 시작 화면에서 "바로 시작" 탭 → "동물원" 장면 일러스트 등장
- 그림 속 사자/기린/원숭이를 보고 하단 도크에서 음절 블록을 골라 단어 슬롯에 채움
- 완성된 단어 칩을 그림 속 해당 객체로 드래그 → 칭찬 사운드 + TTS 발음

### S2. 부모와 함께 놀이
- "오늘은 비 오는 날 단어 해보자" — 부모가 카테고리 "자연/날씨" 선택
- 코칭 프롬프트 "이 그림에 뭐가 있을까?" 가 화면 상단에 노출됨 → 부모가 그대로 읽어줌
- 아이가 망설이면 자동 힌트(첫 음절 강조) 제공, 부모는 "잘했어!" 반응

### S3. 교실 보조 도구
- 교사가 태블릿을 큰 화면에 미러링, "오늘은 가을 풍경"
- 카테고리 "자연·계절" 한정, 5문제 짧은 세션
- 한 명씩 돌아가며 음절 블록을 탭, 전원 따라 발음

### S4. 한국어 학습자 셀프 학습
- "고유어" 카테고리 전체, 20문제, TTS 자동 재생 ON
- 한자어 대신 순우리말(예: "하늘", "바람", "비")만 노출되어 어감을 학습

## 6. 데이터 요구사항

### 6.1 단어 카테고리 (고유어 우선)

| 카테고리 | 예시 단어 | 최소 보유 |
|---------|----------|----------|
| 자연 | 하늘, 바람, 구름, 나무, 별빛, 비, 해, 달 | 12개 |
| 동물 | 강아지, 고양이, 토끼, 다람쥐, 새, 곰 | 12개 |
| 일상 사물 | 우산, 신발, 가방, 우유, 책, 공 | 12개 |
| 행동 | 달리다, 먹다, 자다, 웃다, 울다 | 8개 |
| 감정(체언) | 기쁨, 슬픔, 무서움, 설렘 | 6개 |

> MVP 출시 기준 단어 총량 **50개 이상**, 장면당 정답 1~3개 매핑 가능해야 함.

### 6.2 장면(일러스트) 요구사항

| 항목 | 현황 / 요구사항 |
|------|---------|
| 형식 | 현재: Unsplash 실사 URL(`bgImage`) + 이모지 콜라주(`bgEmojis`). Phase 4에서 자체 SVG/WebP로 교체 |
| 매핑 | 장면 1개 = 정답 단어 1~3개 + 각 단어의 hotspot 좌표(상대값 0~1) |
| 카테고리 정합 | 한 장면의 정답들은 동일/인접 카테고리에서 추출 |
| 출시 기준 | ✅ 장면 8개 완성 (자연 3 / 동물 3 / 일상 2) — playground 단어 보강 필요 |

### 6.3 단어 데이터 스키마 (잠정)

```js
{
  word: '우산',
  syllables: ['우', '산'],
  category: '일상',
  illustration: 'rain.svg',     // 선택: 단독 이모지로 폴백 가능
  hotspot: { x: 0.42, y: 0.65, r: 0.08 }, // 정답 객체 좌표(상대값)
  hint: '비 올 때 쓰는 것',
  emoji: '☂️'                    // 일러스트 실패 시 폴백
}
```

`1_chosung_quiz/src/data/words.js` 의 `{ emoji, word, category, imageUrl }` 구조 상위 호환.

## 7. UX / 모바일 우선 요구사항

| 항목 | 권장 사양 |
|------|----------|
| 최소 터치 타겟 | **56×56dp** (음절 블록), 정답 객체 hotspot은 ±20dp 자성 스냅 |
| 화면 모드 | **세로(Portrait) 우선** — 일러스트 + 도크 수직 |
| 폴백 모드 | 가로일 경우 좌(일러스트) ↔ 우(도크) 분할 |
| 레이아웃 | 상단 60% 일러스트 / 중간 15% 단어 슬롯 / 하단 25% 음절 도크 |
| 입력 방식 | 탭 + 드래그 모두 지원 (탭 시 다음 빈 슬롯 자동 배치) |
| 햅틱 | `navigator.vibrate(20)` (Android), iOS는 무시 (graceful) |
| TTS 정책 | 사용자 탭 직후에만 트리거 (자동재생 정책) |
| 폰트 | Jua / Gowun Dodum (1단계와 통일), `font-display: swap` |
| 접근성 | 다크 모드, 폰트 크기 조절, 색상 대비 WCAG AA 이상 |

## 8. 비기능 요구사항

| 항목 | 요구사항 |
|---|---|
| 성능 | 장면 전환 < 500ms, 음절 탭 응답 < 100ms, 60fps 애니메이션 |
| 호환성 | iOS Safari 15+, Chrome (Android 9+), 데스크톱 Chrome/Edge/Safari 최신 |
| 반응형 | 320px ~ 1024px (스마트폰 ~ 태블릿 우선) |
| 오프라인 | 초기 로드 후 네트워크 없이도 동작 (일러스트 캐싱) |
| 프라이버시 | 외부 서버 전송 없음. 모든 진척도는 localStorage / IndexedDB 로컬 |
| 의존성 | npm 의존성 0 (Vanilla JS + CSS) — 1단계 기조 유지 |
| 데이터 한도 | localStorage 5MB 한도 주의, 누적 학습 데이터는 IndexedDB로 분리 |

## 11. 홈·설정·완료 화면 디자인 공통 규격

시리즈 전체의 통일감을 위해 **홈 화면(시작 화면)**, **설정 화면**, **게임 완료 화면(end-screen)** 은 `1_chosung_quiz` 의 디자인 시스템을 기준으로 제작한다.

| 디자인 요소 | 규격 |
|---|---|
| 제목 폰트 | **Jua** (Google Fonts) |

| 버튼 디자인 | 1단계 `components.css` 버튼 스타일 — 큰 라운드 버튼, CSS 변수 색상 |

| 배경 | `--color-bg` 등 공통 토큰과 동일한 배경 색상 |

> 플레이 화면(장면 일러스트·음절 도크) 등 게임 고유 콘텐츠 화면은 각 게임 특성에 맞게 확장 가능하다.  
> 단, 시작·설정·완료 화면은 공통 디자인 언어를 유지한다.
