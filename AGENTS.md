# 3_word_network — 순우리말 어휘망 연결 게임

## Purpose
음절 단위 읽기가 가능해진 학습자에게 친숙한 고유어(순우리말) 어휘를 맥락 그림과 함께 제시하여, 음절들을 수평적으로 연결해 의미 있는 단어를 생성하는 능력을 기른다. 추상적 한자어로 진입하기 전, 일상 어휘 기반의 '읽기 자동화'를 완성하는 단계.

## Target & Cognitive Goal

| 항목 | 내용 |
|------|------|
| 대상 연령 | 만 5 ~ 7세 (유아 후기 ~ 초등 1학년) |
| 발달 단계 | 어휘 확장 + 수평적 읽기 |
| 핵심 인지 목표 | 시각적 맥락 ↔ 어휘 매칭, 음절 시퀀스 조립, 체언 우세 어휘 발달 |
| 선행 게임 | `../2_syllable_assembly/` — 음절 조립 능력 보유 |
| 후행 게임 | `../4_morpheme_detective/` — 단어 속 의미 단위 인지로 진입 |

## Game Mechanics

### 핵심 루프
1. 동화책 한 장면 또는 일상 상황 일러스트 제시 (예: "비 오는 날", "동물원 풍경")
2. 음절 블록들이 섞여 있는 하단 영역에서 알맞은 블록을 순서대로 선택
3. 선택된 블록이 '기차 칸' 또는 '징검다리'로 연결되어 단어 완성
4. 완성된 단어를 그림 속 객체에 드래그하여 매칭 (예: "우산" → 그림 속 우산)
5. TTS로 단어 발화 + 칭찬 사운드

### 어휘 카테고리 (고유어 우선)
| 카테고리 | 예시 단어 |
|---------|----------|
| 자연 | 하늘, 바람, 구름, 나무, 별빛 |
| 동물 | 강아지, 고양이, 토끼, 다람쥐 |
| 일상 사물 | 우산, 신발, 가방, 우유 |
| 행동 | 달리다, 먹다, 자다, 웃다 |
| 감정 (체언) | 기쁨, 슬픔, 무서움 |

### 적응형 학습 로직
- 학습자가 3회 이상 망설이면 자동으로 첫 음절 힌트 제공
- 부담 신호(연속 오답, 긴 휴지) 감지 시 놀이 비중 자동 증가
- 학부모 코칭용 프롬프트 텍스트 게임 내 표시 (예: "아이에게 '이 그림에 뭐가 있을까?' 라고 물어보세요")

## Mobile-First Considerations

| 항목 | 권장 사양 |
|------|----------|
| 최소 터치 타겟 | **56×56dp** (음절 블록) |
| 화면 모드 | **세로(Portrait) 우선** — 일러스트 + 블록 도크 수직 레이아웃 |
| 폴백 모드 | 가로일 경우 좌(일러스트) ↔ 우(블록 도크) 분할 |

### 모바일 레이아웃 전략
- **상단 60%**: 일러스트 영역 (탭 가능 객체 영역에 미묘한 발광)
- **중간 15%**: 정답 단어가 채워지는 슬롯 (기차/징검다리 비주얼)
- **하단 25%**: 음절 블록 도크 — 가로 스크롤로 더 많은 블록 표시
- **드래그 + 탭** 모두 지원 — 음절 블록을 탭하면 다음 빈 슬롯으로 자동 배치

### 인터랙션 보완
- 탭 시 **햅틱 피드백**(`navigator.vibrate(20)`) — 안드로이드만, iOS Safari는 무시됨
- TTS 발화는 **사용자 탭 직후**에만 트리거 (자동재생 정책 회피)
- 일러스트는 SVG 우선 (망점·해상도 자유) — 못 지원시 WebP 폴백

### 성능
- 일러스트는 **lazy-load**, 다음 스테이지 1개 미리 fetch
- 한 화면에 동시 노출되는 음절 블록은 최대 12개 (선택지 과부하 방지 + 모바일 가독성)

## For AI Agents

### Working In This Directory
- 미구현 설계 단계 — 본 AGENTS.md를 PRD로 활용
- 권장 스택: Vanilla JS + CSS (`1_chosung_quiz` 패턴 재사용), 포트 **3003**
- 단어 데이터는 `1_chosung_quiz/src/data/words.js` 구조 참조 가능 (emoji + word + category)

### Implementation Priorities
1. **그림-단어 매칭 데이터셋** — 카테고리별 일러스트 + 정답 단어 매핑
2. **음절 블록 시퀀스 검증** — 정답 음절 순서, 부분 정답 처리
3. **드래그 앤 드롭** — 2단계 게임과 컴포넌트 공유 가능 (라이브러리화 검토)
4. **TTS** — `1_chosung_quiz/src/js/tts.js` 패턴 재사용 (Web Speech API ko-KR)
5. **반응형 뷰포트** — 세로 ↔ 가로 자동 레이아웃 분기

### Common Patterns to Adopt
- 단어 데이터 구조: `{ word, syllables: ['우', '산'], category, illustration: 'rain.svg', hint }`
- 카테고리 필터링은 `1_chosung_quiz/src/js/settings.js`의 `filterWords()` 패턴 참조
- localStorage 영속화 분리: `storage.js` 단일 책임 원칙

### Testing Requirements
- 한글 IME가 끼어들지 않도록 입력은 **블록 선택 방식**으로 한정 (직접 타이핑 X)
- 일러스트 로드 실패 시 이모지 폴백 (`1_chosung_quiz`의 `img.onerror` 패턴 재사용)
- iPad / 갤럭시 탭 / 보급형 안드로이드 실기기 테스트
